Every meaningful work relationship depends on shared context. Project memory is how AdeHQ builds that context for your AI employees — a persistent, structured store of the decisions made, facts agreed upon, and important background that lives inside each project room. Your AI employees reference and update it continuously, so every session starts exactly where the last one ended.

## Why Project Memory Matters

Without persistent memory, every new session with an AI is a blank slate. You repeat background information, re-explain constraints, and re-establish decisions that were already made. Project memory eliminates that friction entirely.

  <Card title="No Repeated Briefings" icon="circle-check">

  </Card>
  <Card title="Consistent Context Across the Team" icon="users">

## What Gets Stored in Project Memory

Project memory captures the information that matters most across sessions. Common categories include:

<Accordion title="Decisions">
  Key choices that have been made and should not be revisited unless explicitly changed — for example, "We are targeting SMB customers only" or "The API will use REST, not GraphQL."
</Accordion>

<Accordion title="Facts and Background">
  Stable context your AI employees need to do their job accurately — brand voice guidelines, technical constraints, stakeholder names, product details, and similar reference material.
</Accordion>

<Accordion title="Agreed-Upon Priorities">
  The current ranking of tasks, features, or initiatives so AI employees always know what to focus on without asking each session.
</Accordion>

<Accordion title="Progress Markers">
  Records of what has been completed, what is in progress, and what is blocked — giving AI employees an accurate picture of where the work stands.
</Accordion>

## How AI Employees Use Memory

AI employees don't just passively read from memory — they actively engage with it throughout their work.

<Steps>
  <Step title="Reading on Session Start">
    When you open a project room, your AI employees load the current project memory and use it to contextualize everything that follows in the session.
  </Step>
  <Step title="Referencing During Tasks">
    As they work, AI employees pull relevant memory entries to inform their responses, decisions, and actions — the same way a human colleague recalls past agreements.
  </Step>
  <Step title="Updating as Work Progresses">
    When a new decision is made or important context emerges, the AI employee adds or updates the relevant memory entry to keep the record current.
  </Step>
</Steps>

<Note>
  AI employees surface which memory entries they're drawing on when relevant, so you can always trace why they made a particular recommendation or took a specific action.
</Note>

## Reviewing and Editing Project Memory

You stay in control of your project memory at all times. Every entry is visible, editable, and removable directly from the project room.

<Steps>
  <Step title="Open Memory in Your Project Room">
    Navigate to the project memory panel inside any project room to see all current entries organized by category.
  </Step>
  <Step title="Review Existing Entries">
    Read through stored decisions, facts, and context. Each entry shows when it was created or last updated and which AI employee recorded it.
  </Step>
  <Step title="Edit or Remove Entries">
    Update outdated information or remove entries that are no longer relevant. Changes take effect immediately — AI employees use the updated memory in their next interaction.
  </Step>
  <Step title="Add Entries Manually">
    You can add memory entries yourself at any time, not just when an AI employee records them. This is useful for seeding a new project room with critical background before work begins.
  </Step>
</Steps>

<Tip>
  Seed a new project room with a handful of key memory entries before your first session. A brief summary of the project goal, the target audience, and any hard constraints gives your AI employees a strong starting point right away.
</Tip>

<Warning>
  Deleting a memory entry is immediate and permanent within the session. If your AI employees were relying on a removed entry, they will no longer have access to that context. Re-add it manually if needed.
</Warning>

## Memory Is Scoped, Not Shared Globally

Project memory belongs to the room it was created in. Your AI employees do not carry memory from one project room into another, and memory entries cannot be accidentally cross-contaminated between initiatives.

<Info>
  If you need shared context across multiple rooms — such as company-wide brand guidelines — add it manually to the memory of each relevant room, or reference it in a shared document that your AI employees can access via their assigned tools.
</Info>
