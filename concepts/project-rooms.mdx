When you have a goal to pursue — launching a product, resolving a support backlog, drafting a strategy — you open a project room. A project room is a dedicated, persistent workspace that brings together your human teammates and AI employees under one shared context, with a live conversation thread and ongoing project memory.

## What's Inside a Project Room

Every project room contains the tools and context your team needs to move work forward without switching tabs or losing thread.

<CardGroup cols={2}>
  <Card title="Shared Conversation Thread" icon="comments">
    All messages, AI responses, task updates, and approval requests flow through a single chronological thread visible to every room member.
  </Card>
  <Card title="AI Employees" icon="users">
    The AI employees you assign to the room are available immediately, already aware of the project's history and memory.
  </Card>
  <Card title="Project Memory" icon="brain">
    Decisions and key context are stored persistently, so every session picks up exactly where the last one left off.
  </Card>
  <Card title="Real-Time Collaboration" icon="bolt">
    Multiple human teammates can join the same room and interact simultaneously — no waiting, no conflicts.
  </Card>
</CardGroup>

## Creating a Project Room

<Steps>
  <Step title="Open Your Workspace">
    Log in to AdeHQ and navigate to your team workspace. All existing project rooms appear on the workspace dashboard.
  </Step>
  <Step title="Create a New Room">
    Select **New Project Room**, give it a clear name that reflects the initiative (e.g., "Q3 Marketing Campaign" or "Backend API Refactor"), and add an optional description.
  </Step>
  <Step title="Assign AI Employees">
    Choose which AI employees should work in this room. Each employee brings their role, tools, and a fresh memory context scoped to this project.
  </Step>
  <Step title="Invite Human Teammates">
    Add any real team members who need access. They can join and contribute to the conversation in real time as soon as they're invited.
  </Step>
  <Step title="Start Collaborating">
    Send your first message. Your AI employees are ready to receive tasks, answer questions, take actions, and build on the project as it evolves.
  </Step>
</Steps>

## Multiple Rooms for Multiple Initiatives

You are not limited to a single project room. Each room is fully independent — its own conversation history, its own AI employee assignments, and its own project memory.

<Tip>
  Keep rooms focused on a single initiative or objective. Narrow scope means sharper AI employee context and more useful memory, which leads to better results over time.
</Tip>

<Accordion title="Can the same AI employee work in more than one room?">
  Yes. The same AI employee role can be assigned to multiple project rooms. Each room maintains its own isolated memory and conversation history, so the employee's context in one room never bleeds into another.
</Accordion>

<Accordion title="What happens to a room's history if I close it?">
  Project rooms and their full conversation history, memory entries, and file context persist in your workspace. Closing or archiving a room does not delete its contents — you can return to it at any time.
</Accordion>

## Real-Time Collaboration

Project rooms are built on a real-time backend, meaning changes appear instantly for every participant. When a teammate sends a message, when an AI employee completes a task, or when an approval request is raised, every active member of the room sees it immediately — no refresh required.

<Info>
  Real-time sync ensures that your workspace state is always consistent across every participant's screen — no manual refresh needed.
</Info>

## Simulated Calls in a Room

Within any project room, you can escalate from the chat thread to a simulated call with your AI workforce. Calls are great for rapid-fire delegation, brainstorming, or reviewing progress in a more conversational format.

<Note>
  When you start a call, all AI employees assigned to the room join with full awareness of the existing project memory and conversation history — no briefing required.
</Note>
