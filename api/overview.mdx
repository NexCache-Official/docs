## Base URL

```
https://ade-hq-eight.vercel.app/api   # production
http://localhost:3000/api             # local dev
```

All routes use Next.js Route Handlers (`src/app/api/`). There are no Server Actions.

## Authentication

Most endpoints require:

```
Authorization: Bearer {supabase_access_token}
```

Destructive operations additionally require **password re-authentication** in the request body. See [Authentication](/api/authentication).

## Message send response (V16.8+)

`POST /api/rooms/:roomId/messages` returns immediately after queuing â€” AI replies are **not inline**:

```json
{
  "humanMessage": {},
  "queuedRuns": [],
  "blockedRuns": [],
  "collaborationPlan": {},
  "aiResponses": [],
  "aiMessages": []
}
```

The client polls `GET .../agent-runs` and calls `POST .../process` for each run.

## Error format

Structured errors via `message-errors.ts`:

```json
{
  "ok": false,
  "code": "not_room_member",
  "error": "You are not a member of this room.",
  "details": {}
}
```

Special case: status **207** when message saved but AI queue failed (`ai_runtime_failed_but_message_saved`).
