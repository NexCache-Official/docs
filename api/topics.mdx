## PATCH /api/topics/:topicId

Update topic metadata including AI participation mode.

**Auth:** Topic creator or workspace admin. Archive requires admin.

**Request body (all optional):**

```json
{
  "title": "string",
  "status": "active | paused | resolved | archived",
  "priority": "low | normal | high | urgent",
  "summary": "string",
  "aiParticipationMode": "silent_observation | manual_only | smart_assist_lite | smart_assist | active_team"
}
```

General topics store `smart_assist_lite` when UI selects "Smart assist". Non-General topics use `smart_assist`.

**Source:** `src/app/api/topics/[topicId]/route.ts`

---

## DELETE /api/topics/:topicId

Soft-archive a topic. Cannot archive the General topic.

**Auth:** Topic creator or workspace admin.

---

## GET /api/topics/:topicId/ai-control

Read topic AI control metadata. Added in V16.8.

**Auth:** Workspace member with room access.

**Query params:** `workspaceId`, `roomId` (optional — inferred from topic row)

**Response:**

```json
{
  "metadata": {
    "aiParticipationMode": "smart_assist",
    "aiStopped": false,
    "smartAssistPaused": false,
    "aiPausedUntil": null,
    "blockedEmployeeIds": []
  }
}
```

---

## POST /api/topics/:topicId/ai-control

Stop, pause, or resume AI in a topic. Added in V16.8.

**Auth:** Workspace member with room access.

**Request body:**

```json
{
  "action": "stop_all | resume | pause_smart | stop_employee | resume_employee",
  "employeeId": "string (required for stop/resume employee)",
  "pauseMinutes": 60
}
```

| Action | Effect |
|--------|--------|
| `stop_all` | Sets `aiStopped: true`, cancels queued/running runs |
| `resume` | Clears all AI blocks and blocked employees |
| `pause_smart` | Pauses smart assist for N minutes |
| `stop_employee` | Adds employee to `blockedEmployeeIds` |
| `resume_employee` | Removes employee from block list |

**Response:** `{ "topic": { ...updated topic... } }`

**Source:** `src/app/api/topics/[topicId]/ai-control/route.ts`

## POST /api/topics/:topicId/members

Add an AI employee to a topic.

**Request body:**

```json
{
  "employeeId": "string (required)",
  "notificationLevel": "all | mentions | none"
}
```

---

## POST /api/topics/:topicId/read

Update read cursor for unread tracking.

**Request body:**

```json
{
  "lastReadMessageId": "uuid (required)"
}
```

---

## POST /api/topics/:topicId/summarize

Generate LLM summary of recent topic messages.

**Request body:**

```json
{
  "saveToMemory": true
}
```

## Participation modes reference

| Mode | Default for | Description |
|------|-------------|-------------|
| `smart_assist_lite` | General topic | Light ambient responses, greetings |
| `smart_assist` | Other topics | Full smart participation |
| `active_team` | — | Aggressive heuristics |
| `manual_only` | — | @mentions and DMs only |
| `silent_observation` | — | AI observes, doesn't respond |

Effective mode respects runtime controls (`aiStopped`, `smartAssistPaused`). See `src/lib/topic-ai-control.ts`.
