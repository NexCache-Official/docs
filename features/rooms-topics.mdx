## Project rooms

Two kinds of rooms, enforced at the database level (V16.7):

| Kind | Description | `dm_employee_id` | Created by |
|------|-------------|------------------|------------|
| `channel` | Group collaboration space | Must be null | Create room modal |
| `dm` | 1:1 with an AI employee | Required | Hiring an employee |

DB constraint `project_rooms_kind_shape` in migration `20250630120000`.

Helpers in `src/lib/rooms.ts`:

```typescript
isGroupChannel(room)  // kind !== "dm"
isDirectMessage(room) // kind === "dm"
getGroupChannels(rooms)
getDirectMessages(rooms)
```

### UI navigation (V16.7)

- **Sidebar** — separate sections for Channels and Direct Messages
- **Rooms page** — filtered lists
- **Command bar** — distinct actions for "New channel" vs employee DMs

### Room membership

- Workspace admins bypass membership checks
- Send permission: `assertCanSendRoomMessage()`

## Topics

Every room has at least one topic:

| Room kind | Default topic label |
|-----------|---------------------|
| Channel | **General Chat** |
| DM | **Direct Chat** |

Helper: `mainChatLabel(isDm)` in `src/lib/topics.ts`

### Topic fields

| Field | Description |
|-------|-------------|
| `title` | Display name (unique per room) |
| `status` | `active`, `paused`, `resolved`, `archived` |
| `priority` | `low`, `normal`, `high`, `urgent` |
| `summary` | LLM-generated or manual |
| `metadata` | AI participation mode, control flags |
| Counters | `message_count`, `agent_run_count`, etc. |

### Archive and restore (V17)

- Archive via DELETE `/api/topics/:id` or topic panel
- **Restore** archived topics from the topic panel (V17)
- General/Direct Chat topic cannot be archived

## AI participation modes

| Mode | Default for | Behavior |
|------|-------------|----------|
| `smart_assist_lite` | General/Direct Chat | Light ambient — greetings, role match |
| `smart_assist` | Other topics (V17 default on create) | Full smart participation |
| `active_team` | — | Aggressive heuristics |
| `manual_only` | — | @mentions and DMs only |
| `silent_observation` | — | AI observes, doesn't respond |

### Runtime AI controls (V16.8)

Stored in `topic.metadata`, managed via topic panel (V17: collapsible section):

| Control | Metadata key | Effect |
|---------|--------------|--------|
| Stop all AI | `aiStopped` | Block all queueing, cancel active runs |
| Pause smart assist | `smartAssistPaused` + `aiPausedUntil` | Temporary smart assist off |
| Block employee | `blockedEmployeeIds[]` | Prevent specific employee from responding |

API: `POST /api/topics/:topicId/ai-control`

Effective mode: `getEffectiveParticipationMode()` in `src/lib/topic-ai-control.ts`

## API endpoints

- `GET/POST /api/rooms/:roomId/topics`
- `GET /api/rooms/:roomId/topics/:topicId/agent-runs` — run polling
- `PATCH/DELETE /api/topics/:topicId`
- `GET/POST /api/topics/:topicId/ai-control`
- `POST /api/topics/:topicId/members|read|summarize`
