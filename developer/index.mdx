## Current platform status

| Area | Status | Notes |
|------|--------|-------|
| Auth & onboarding | Shipped | Email/password via Supabase, multi-workspace |
| Rooms & topics (Messaging v2) | Shipped | Zulip-style threads, AI participation modes |
| AI runtime | Shipped | SiliconFlow-only live provider, cost guardrails |
| Workforce | Shipped | Hire, configure, mention in rooms |
| Work graph | Shipped | Tasks, memory, approvals, work log linked to agent runs |
| Demo mode | Dev-only | Gated by `NEXT_PUBLIC_ENABLE_DEMO_MODE` |
| Real integrations | Not built | Slack, GitHub, billing, MCP, real calls |

## How to use these docs

<Steps>
  <Step title="Start with PRDs for product intent">
    Read the [Messaging v2 PRD](/prds/messaging-v2) and [AI runtime PRD](/prds/ai-runtime) to understand *why* features exist before diving into implementation.
  </Step>
  <Step title="Trace a message end-to-end">
    Follow the flow in [Messaging](/features/messaging) from `RoomChat` → API → agent run → SiliconFlow reply.
  </Step>
  <Step title="Look up endpoints">
    Use the [API reference](/api/overview) when wiring new clients or debugging production.
  </Step>
  <Step title="Check schema changes">
    Every migration is listed in [Database migrations](/database/migrations) with what it adds.
  </Step>
</Steps>
