## V16.2 — Workspace owner delete

```sql
create policy "workspaces_delete_owner"
on public.workspaces for delete
using (owner_id = auth.uid());
```

Enables permanent workspace deletion via `DELETE /api/workspaces/:id`.

## V16.4 — Tools catalog seed

Idempotent insert of 23 global tools into `public.tools`. Required before `workspace_tools` and `employee_tools` FK references work during onboarding.

Tools include: web-search, github, notion, slack, siliconflow, vercel, supabase, and others (mostly `mock` or `not_connected` status).

## V16.7 — Room kind constraints

```sql
ALTER TABLE public.project_rooms
  ADD CONSTRAINT project_rooms_kind_shape CHECK (
    (kind = 'dm' AND dm_employee_id IS NOT NULL)
    OR (kind <> 'dm' AND dm_employee_id IS NULL)
  );
```

Enforces: DMs always have a counterpart employee; channels never do.

## V16.8 — Channel AI orchestration

**New columns on `agent_runs`:**
- `response_reason text` — why this employee was selected
- `root_trigger_message_id text` — links follow-ups to original human message
- `run_metadata jsonb` — collaboration plan, role, status

**New column on `ai_employees`:**
- `participation_style text default 'balanced_teammate'`

**Data migration:** General topics get `aiParticipationMode: smart_assist_lite`.

**Indexes:**
- `idx_agent_runs_root_trigger`
- `idx_agent_runs_topic_status`

## V16.9 — Conversation orchestrator

**New column on `agent_runs`:**
- `depends_on_run_id text` — collaborator waits for lead run

**Indexes:**
- `idx_agent_runs_depends_on`
- `idx_agent_runs_collab_lookup`
